# Case study source notes

Author-provided context for portfolio copy. **Not imported at build time** — used as the source of truth when editing `../workflows.ts`, `../outputs.ts`, and `../reconstruction.ts`.

## Suggested template per study

Create one markdown file per case study (any filename; avoid `#` in paths if possible):

1. **Product / context** — what the product was, who used it, why it mattered
2. **Your role and constraints** — responsibilities, collaborators, technical or org limits
3. **One before / after moment** — concrete shift in user behavior or system behavior
4. **What you want a hiring manager to remember** — 3–5 proof points, not buzzwords

Optional sections for major visuals (e.g. challenge diagram, interactive prototype): same four blocks scoped to that artifact.

## Workflow

1. Add or update markdown here
2. Rewrite the matching `content/case-studies/<slug>.ts` from these notes
3. Preview at `/work/<slug>`
4. Note progress in `docs/refactor-status.md`
